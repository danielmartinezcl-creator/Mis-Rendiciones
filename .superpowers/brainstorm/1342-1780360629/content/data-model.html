Diseño — Sección 2: Modelo de datos Supabase

Schema multi-tenant con soporte nativo para N niveles de aprobación. Cada tabla tiene org_id para aislar datos entre empresas clientes.

🏢 organizations
🔑iduuid PK
nametext
slugtext unique
countrytext default 'CL'
currencytext default 'CLP'
plan'free'|'pro'|'enterprise'
logo_urltext
👤 users (extiende auth.users)
🔑iduuid → auth.users
🔗org_iduuid → organizations
full_nametext
role'admin'|'approver'|'employee'
can_submitbool default true
can_approvebool default false
departmenttext
bank_accounttext (reembolso)
⚙️ approval_policies
🔑iduuid PK
🔗org_iduuid → organizations
nametext
levelsjsonb ← clave multi-nivel
is_defaultbool
levels: [{level:1, approver_id:"..."}, {level:2, approver_id:"..."}]
En v1 solo se usa level:1. Level:2 se activa en v2 con UI.
📋 expense_reports
🔑iduuid PK
🔗org_iduuid → organizations
🔗submitter_iduuid → users
titletext
status'draft'|'submitted'|'pending_l2'|'approved'|'partially_approved'|'rejected'|'reimbursed'
current_levelint default 1
total_amountnumeric
approved_amountnumeric
currencytext default 'CLP'
reimbursed_attimestamptz
🧾 expense_items
🔑iduuid PK
🔗report_iduuid → expense_reports
descriptiontext
amountnumeric
currencytext (CLP/USD/EUR)
amount_clpnumeric (convertido)
datedate (fecha del gasto)
🔗category_iduuid → expense_categories
merchanttext (proveedor)
doc_type'boleta'|'factura'|'factura_exenta'|'ticket'|'otro'
doc_numbertext
status'pending'|'approved'|'rejected'
rejection_reasontext
ocr_rawjsonb (respuesta Claude)
📎 attachments
🔗item_iduuid → expense_items
storage_pathtext (Supabase Storage)
file_type'image'|'pdf'
thumbnail_pathtext
📜 expense_report_approvals (audit log)
🔗report_iduuid → expense_reports
🔗approver_iduuid → users
levelint (1, 2, ...)
action'approved'|'rejected'|'returned'
items_approveduuid[] (IDs aprobados)
items_rejecteduuid[] (IDs rechazados)
notestext
created_attimestamptz
RLS (Row Level Security) en cada tabla: Los usuarios solo ven registros de su org_id. Los rendidores solo ven sus propias rendiciones. Los aprobadores ven rendiciones de empleados asignados. El admin ve todo dentro de su org.
