Arquitectura del sistema — Elegí el enfoque

Esta decisión define el stack completo, la velocidad de desarrollo y la escalabilidad del producto.

OPCIÓN A — TODO EN SUPABASE

React + Vite + Supabase All-in

Frontend SPA + Supabase hace todo el backend: Auth, DB, Storage, Edge Functions para OCR y emails.

📱 Frontend — Vercel
React 18 + Vite · React Router · Tailwind CSS · PWA
↕
⚡ Supabase Edge Functions
OCR con Claude · Emails con Resend · Webhooks
↕
🗄️ Supabase
PostgreSQL + RLS · Auth · Storage (fotos)

✓ Pros

  • Un solo proveedor cloud
  • Fácil de mantener
  • Costo muy bajo
  • Desplegable en minutos

✗ Contras

  • Edge Functions en Deno (no Node)
  • Dependencia total de Supabase
OPCIÓN B — NEXT.JS FULL-STACK RECOMENDADO

Next.js 15 + Supabase + Vercel

Frontend + API en un solo proyecto Next.js. Server Actions para OCR, autenticación segura, SSR para rendimiento.

📱 Next.js App Router — Vercel
RSC + Client Components · Tailwind · PWA via next-pwa
↕
🔧 Next.js API Routes / Server Actions
OCR Claude Vision · Emails Resend · Webhooks · Lógica de negocio
↕
🗄️ Supabase
PostgreSQL + RLS · Auth · Storage (fotos boletas)
¿Por qué Next.js para SaaS?
API Routes corren en Node.js (no Deno) → fácil integrar cualquier SDK. Server Components mejoran performance en móvil. Vercel es el hábitat natural de Next.js.

✓ Pros

  • Stack estándar de la industria SaaS
  • Lógica de negocio en el servidor
  • SEO + performance superiores
  • Escala a millones de usuarios

✗ Contras

  • Más complejo que Vite SPA
  • Curva de aprendizaje si lo tocás directamente
OPCIÓN C — REACT + API SEPARADA

Vite SPA + Node.js Backend

Frontend React en Vercel, backend Express/Hono en Railway o Fly.io. Máxima flexibilidad pero más infra.

📱 React + Vite — Vercel
SPA · Tailwind · React Query
↕ REST API
🖥️ Node.js / Hono — Railway
OCR · Emails · Business logic · Webhooks
↕
🗄️ Supabase
PostgreSQL + RLS · Auth · Storage

✓ Pros

  • Máxima flexibilidad
  • Escala backend independiente

✗ Contras

  • 2 deploys separados
  • CORS, auth entre servicios
  • Costo extra (Railway ~$5-20/mes)
  • Más complejo sin beneficio claro vs B
